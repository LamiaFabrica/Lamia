## 3. Language Specification

### 3.1 Syntax Structure
```lamia
manifest <name>(<parameters>) -> <return_type> @<mode> {
    <statements>
}
```

### 3.2 Key Keywords
- `manifest` - Function definition
- `create` - Object instantiation
- `crystal` - Universal return type
- `RADIANT_` - Prefix for luminous constructs
- `@ludicrous` - Performance mode annotation

### 3.3 Performance Modes
- `@standard` - Default execution
- `@ludicrous` - Maximum performance (3x speed)
- `@quantum` - Quantum computing preparation
- `@embedded` - Microcontroller optimization
- `@realtime` - Real-time guarantees

### 3.4 Type System
- **RADIANT** - String type (radiant = bright/clear text)
- **SHIMMER** - Number type (shimmer = fluid mathematics)
- **LUMINA** - Boolean type (lumina = light/dark)
- **CONSTELLATION** - Array type (ordered collection)
- **NEBULA** - Object/Map type (key-value cloud)
- **GALAXY** - Function type (executable system)
- **CRYSTAL** - Universal container type
